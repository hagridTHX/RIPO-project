PURPOSE:
Initialize gesture controller with debounce timers and gesture history tracking to prevent
accidental repeated actions from jittery gesture recognition.

WHAT IT DOES:

• Line 6: Sets PyAutoGUI.FAILSAFE to False
  - Allows script to control mouse/keyboard without failsafe (corner trigger)
  - Normally True for safety, but disabled here for uninterrupted gesture control
  
• Lines 9-10: Click debounce setup
  - last_click_time: Tracks timestamp of last mouse click performed (Line 9)
  - click_cooldown: Sets minimum 1.0 second between consecutive clicks (Line 10)
  - Prevents accidental double-clicks from gesture jitter
  
• Lines 12-13: Scroll debounce setup
  - last_scroll_time: Tracks timestamp of last scroll action (Line 12)
  - scroll_cooldown: Sets minimum 0.05 second between consecutive scrolls (Line 13)
  - Allows fast scrolling (20 scrolls per second at 60 FPS) while preventing spam
  
• Lines 15-16: Swipe debounce setup
  - last_swipe_time: Tracks timestamp of last swipe gesture (Line 15)
  - swipe_cooldown: Sets minimum 1.0 second between consecutive swipes (Line 16)
  - Prevents accidental repeated page navigation from motion blur
  
• Lines 18-19: Logging cooldown setup
  - last_log_time: Tracks when status was last printed (Line 18)
  - log_cooldown: Prints status only every 1.0 second (Line 19)
  - Reduces console spam during continuous thumb gestures
  
• Line 21: Initializes wrist_history as empty list
  - Stores tuples of (timestamp, wrist_x_coordinate)
  - Used to calculate hand velocity for swipe detection
  - Tracks hand position over time (typically last 0.4 seconds of data)

KEY CHARACTERISTICS:
• All debounce timing values fine-tuned for 60 FPS operation
• Scrolling has shortest cooldown (0.05s) allowing greatest responsiveness
• Clicks and swipes have longest cooldown (1.0s) preventing accidental repetition
• Wrist history enables velocity-based motion detection instead of simple position
